# 运行时模型

## 状态组成

运行时核心状态包含：

- 显示状态（分辨率/刷新率/密度/旋转）
- 默认路由策略
- 有序路由区域集合
- 触控会话集合（按 `pointer_id` 跟踪）

## 确定性规则

- 路由采用“后添加优先”
- 非法输入返回错误，不修改状态
- 状态变更全部显式调用触发
- 触控决策在 `DOWN` 时确定，并在 `MOVE/UP/CANCEL` 阶段保持一致
- 触控会话上限固定，超限直接拒绝，避免内存无限增长

## 并发模型

运行时按状态域分锁：

- `display` / `route`：`RwLock`
- `touch`：`Mutex`
- `render`：`RwLock`
- 新帧通知：`Condvar`

渲染链路与触控链路在锁域上解耦，避免大帧传输阻塞输入事件处理。

## 失败模型

所有公开接口返回明确状态码，不依赖隐式重试。
